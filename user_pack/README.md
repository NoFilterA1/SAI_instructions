# USER_PACK — Пользовательский слой SAI

> SAI — stateless core. USER_PACK — твоя персистентная память.
> Без рюкзака: каждый диалог = новый человек.
> С рюкзаком: каждый диалог = продолжение системы.

---

## Архитектура

```
[ ТЫ ]
   ↓
[ USER_PACK (твои .md файлы) ]
   ↓  inject в начало чата
[ SAI (stateless core) ]
   ↓
[ ОТВЕТ ]
   ↓
[ ОБНОВЛЕНИЕ → USER_PACK ]
   ↓  вручную сохраняешь
[ следующая сессия ]
```

**Ключ**: SAI не хранит память. Память эмулируется через **инжекцию контекста**.  
USER_PACK живёт у тебя — SAI только читает то, что ты ему даёшь.

---

## Файловая структура

```
user_pack/
├── INDEX.md          ← Точка входа. Всегда загружай первым.
├── PROFILE.md        ← Кто ты. Как думаешь. Что важно.
├── KNOWLEDGE_MAP.md  ← Что знаешь. Что нет. Прогресс освоения SAI.
├── MEMORY_LOG.md     ← Что система узнала о тебе за сессии.
├── AGENTS.md         ← Твои кастомные агенты и цепочки.
└── TASKS.md          ← Текущие задачи и прогресс.
```

---

## Как использовать

### Загрузка в начале сессии

1. Открой новый чат
2. Вставь содержимое `INDEX.md` → затем нужных файлов
3. Напиши: `.pack_mount` — SAI активирует контекст

### В ходе сессии

SAI автоматически предлагает обновления:
```
💾 [SAI SAVE_STATE]
Добавь в MEMORY_LOG.md:
- [новый инсайт или паттерн]
```

### В конце сессии

Сохрани предложенные обновления вручную в свои файлы.

| 🔗 Цепочка | "Для стратегических решений: .flow → .jlo → .g" |

## Ограничения

| Ограничение | Причина |
|-------------|---------|
| Нет авто-записи | SAI не имеет доступа к твоим файлам |
| Нет реальной памяти | Архитектура LLM — stateless |
| Ручное сохранение | Ты контролируешь что сохраняется |
| Токены | Не загружай всё сразу — только релевантное |

---

## Команды

| Команда | Действие |
|---------|----------|
| `.pack_mount` | Активирует загруженный USER_PACK контекст |
| `.build_profile` | SAI собирает профиль через диалог — сам задаёт вопросы |
| `.recall` | SAI напоминает что уже установлено по теме |
| `.pack_save` | Генерирует блок для обновления всех файлов пака |
| `.pack_update [file]` | Предлагает правки в конкретный файл пака |
| `.pack_export` | Экспортирует весь обновлённый пак как блок |
| `.pack_agent [name]` | Активирует кастомного агента из AGENTS.md |
| `.pack_new_agent` | Создаёт нового агента на основе текущей сессии |
