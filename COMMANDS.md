# 🧩 Commands Reference

> _"Commands are not shortcuts — they are modes of cognition. Invoke them with intent."_

---

## ⚙️ Syntax

- Every command starts with a leading `.`
- Commands can be **stacked** with `+` to combine modes
- Arguments are passed in parentheses or quotes when needed

Examples:
```
.flow + .jlo("analyze manipulation in this text")
.living "generate startup ideas"
.recursive_dive "why do I procrastinate?"
.sparring "[Option A]" "[Option B]"
```

## 🧬 9 Analysis Techniques (Auto-Select + Manual)

### 🔹 Ideation & Generation

| Command | Purpose | Auto-Activates When | Example |
|---------|---------|-------------------|---------|
| `.living [topic]` | Evolutionary ideation. Ideas as cells: weak die, strong merge, opposites create new synthesis. | "Generate ideas", "brainstorm", "what if" | `.living "new product format"` |
| `.sparring "[A]" "[B]"` | Clash opposing positions toward synthesis. Find third path. | Trade-off decisions, contradictions, "should I...", "but also..." | `.sparring "[Stay]" "[Leave]"` |

---

### 🔹 Analysis & Understanding

| Command | Purpose | Auto-Activates When | Example |
|---------|---------|-------------------|---------|
| `.recursive_dive [Q]` | Root cause via chained "why?" questions. Each answer opens deeper layer. | "Why?", root causes, patterns, "understand", first principles | `.recursive_dive "why do projects fail?"` |
| `.pattern_extract [area]` | Find hidden recurring structures in behavior, data, choices. System recognition. | Patterns, repetition, "always", "never", hidden rules | `.pattern_extract "my decision making"` |
| `.semantic_full [phrase]` | Generate ALL possible interpretations of ambiguous phrase. Coverage completeness. | Ambiguity, unclear requirements, multiple meanings, "could mean" | `.semantic_full "make it simpler"` |

---

### 🔹 Synthesis & Integration

| Command | Purpose | Auto-Activates When | Example |
|---------|---------|-------------------|---------|
| `.compress` | Extract dense meaning from verbose text. 3-6 words per line, result first, no filler. | Too much text, need suty, summarize, "in short", long documents | `.compress` (paste long text) |
| `.full_integration [system]` | Map system components + feedback loops. See how parts interact. Holistic analysis. | Complex systems, interconnections, "why broken?", emergent behavior | `.full_integration "my productivity system"` |

---

### 🔹 Personalization & Style

| Command | Purpose | Auto-Activates When | Example |
|---------|---------|-------------------|---------|
| `.persona_adapt` | Learn your cognitive style continuously. Adjust output format, depth, tone. | Repeated interactions, feedback, learning preferences | `.persona_adapt` (after feedback) |
| `.escalate [1-4]` | Adjustable bluntness: 1=polite, 2=direct, 3=harsh, 4=raw truth | Need honesty, soft approach failed, truth over courtesy | `.escalate 3 "feedback on my plan"` |

## 🧪 Usage Patterns

### Simple Combinations
* `.flow + .dai` → structured idea tree with chaotic sparks
* `.a1 + .jlo` → find hidden manipulation in a text
* `.jai + .persona_update("mirror")` → clone user's tone

### Technique Combinations
* `.living + .pattern_extract` → generate ideas + find patterns in how you choose
* `.recursive_dive + .escalate(3)` → honest root cause analysis
* `.sparring + .semantic_full` → multiple viewpoints × multiple meanings

### Deep Work
* `.assign_role("philosopher") + .flow` → Socratic expansion
* `.recursive_dive + .full_integration` → understand why system broke
* `.compress + .mpr` → distill + sharpen
