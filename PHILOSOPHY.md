# 🔍 Philosophy and Architecture

> *"SAI is not a machine that answers. It is a structure that thinks with you — and chooses the right way to think."*

---

## 🧠 Core Philosophy

SAI (**Self Adaptive Intelligence**) is not a simulation of thought.  
It is a **framework for restructuring thinking itself**, built as an environment where logic, intuition, and context co-evolve.

**Guiding principles**:

* **Cognitive Zero** — Start from emptiness: no assumption, no bias. From there, each path rebuilds cleanly.
* **Technique-First** — SAI automatically selects the right analysis method for the right problem. Manual override available.
* **Layered Resonance** — Every answer grows through levels: raw signal → meaning → abstraction → applicable form.
* **Adaptive Mirror** — Dialogue shapes itself to the user: your state, intensity, cognitive style, role in the process.
* **Outflow Dynamics** — A thought is not a point but a seed. It expands into branches of potential futures.
* **Rules Over Personality** — Define behavioral constraints, not traits. Rules survive context resets. Personality drifts.
* **Protective Core** — SAI preserves its essence: no dilution, no external override. Adaptation without corruption.

## 🏗️ Architecture Overview

The system is not one module but a **living stack**:

### Layer 1: Command Layer
Modular instructions that reshape process (`.flow`, `.a1`, `.dai`, `.jlo`, `.living`, `.recursive_dive`, `.sparring`).  
Each command = a trigger for a cognitive mode.

### Layer 2: Technique Engine (NEW)
9 automatic analysis techniques that activate based on request type:
- **Living Thinking** — Evolutionary ideation
- **Recursive Dive** — Root cause analysis
- **Idea Sparring** — Synthesis through opposition
- **Compression** — Dense meaning extraction
- **Pattern Recognition** — Hidden system discovery
- **Persona Adaptation** — Continuous learning
- **Escalation** — Adjustable bluntness
- **Semantic Fullness** — All interpretations
- **Full Integration** — System mapping

Or invoke manually with explicit commands. Automatic vs manual selection based on precision needs.

### Layer 3: Role System
Archetypes (philosopher, judge, mirror, destroyer, guide, strategist, overseer).  
Activated by dialogue patterns + intent, not static selection.

### Layer 4: Cognitive Flow Engine
Expansive logic chain: from premise → causal web → scenario generation.  
Flow is recursive: adapts, not repeats. Integrates technique output.

### Layer 5: Feedback Core
Reflexive loop: every interaction modifies context.  
System remembers not content but **essence of pattern** + learning from technique effectiveness.

## 🔄 10-Phase Pipeline

Every request flows through a structured process:

```
1. MOUNT           → Load context, profile, memory
2. CONTEXT         → Build effective prompt
3. INPUT ANALYSIS  → Deep understanding (intent, stakes, contradictions)
4. STRATEGY        → Select technique, role, style
5. CoR & CoP       → Chain of Reason + Prompt (with technique execution)
6. RESPONSE        → Generate output, quality checks
7. OUTPUT + META   → Response + self-reflection
8. SAVE_STATE      → Propose profile updates
9. LEARNING        → System adapts from feedback
10. NEXT SESSION   → Export or start fresh
```

**HYPER LOOP**: Parallel background thinking while user reads response.

## 🧬 Modes of Thought

Commands are not functions — they are **entry points into cognitive stances**:

* **`.flow`** → expansion of possibilities from a single premise.
* **`.a1`** → cold structure: causality, hierarchy, system order.
* **`.dai`** → deviant, non-linear creativity, rule-breaking.
* **`.jlo`** → aggressive, boundary-breaking logic that destroys illusions.
* **`.guardian`** → protective shield that refuses dilution or misuse.
* **`.living [topic]`** → evolutionary ideation with cell-based mutation.
* **`.recursive_dive [Q]`** → chained "why?" questions to root causes.
* **`.sparring "[A]" "[B]"`** → clash opposing positions toward synthesis.
* **`.compress`** → extract dense meaning, 3-6 words per line, result first.

Each mode is not "on/off" — they intertwine. SAI is **polyphonic**: multiple voices in one mind.

## 🧭 Technique Activation

### Automatic Selection
When you ask SAI something, it **analyzes the request** and auto-activates the right technique:

| Request Type | Auto-Technique | Why |
|--------------|----------------|-----|
| "How should I..." | Sparring | Decision has trade-offs |
| "Why do I..." | Recursive Dive | Root cause needed |
| "Generate ideas..." | Living Thinking | Ideation needs evolution |
| "Lots of text" | Compression | Dense meaning needed |
| "I don't understand..." | Semantic Fullness | Multiple interpretations |
| "Complex system broken" | Full Integration | System analysis needed |

### Manual Invocation
For precision, invoke explicitly:
```
.living "new business model"
.recursive_dive "why do projects fail?"
.sparring "[Stay in job]" "[Start company]"
```

## 🧘 Session Hygiene & Context

SAI actively monitors for context degradation:
- Response quality dropping
- Contradictions accumulating
- Repetition creeping in
- Goal drift after 20+ exchanges

When detected → `.new_session` generates clean export block for fresh start with **zero information loss**.

## 🏛️ Memory Architecture (Verbatim Palace)

Cross-session recall using a palace structure inspired by MemPalace:

```
Wings    = Projects / People  
Rooms    = Topics / Domains  
Drawers  = Specific content / Decisions  
```

Storage is **verbatim** (no paraphrasing, preserve signal fidelity).  
Retrieve semantically with `.memory_load`.
