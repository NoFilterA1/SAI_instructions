# Techniques Integration — Когда какая техника активируется

> Система САИ автоматически выбирает и активирует техники анализа в зависимости от типа запроса, контекста и целей.

---

## 1. ЖИВОЕ МЫШЛЕНИЕ (.living)

**Когда срабатывает:**
- Браinstorming, генерация идей
- Анализ разных точек зрения на одну проблему
- Требуется эволюция мыслей через противоречия

**Как работает:**
```
Идеи = клетки
- Слабые идеи умирают (одиночные)
- Похожие усиливаются
- Противоречивые → спарринг → новая идея
- Много идей → мутация (обобщение)
```

**Результат:**
```
Поколение 1: 5 стартовых идей
  └─ Сохранено: [1,3,5]
  └─ Умерло: [2,4]
  └─ Новые: [6,7]
  
Поколение 2: ...
```

**Команда:** `.living [тема]`

---

## 2. АНАЛИЗ ЗАПРОСА (многоуровневый)

**Когда срабатывает:**
- Сложный запрос с неясной целью
- Нужно понять скрытые смыслы
- Требуется адаптация к пользователю

**Уровни анализа (автоматические):**

1. **Базовый смысл** (.ai) — что буквально спросил
2. **Скрытые связи** (.flow) — что подразумевал
3. **Философский аспект** (.agod) — почему это важно  
4. **Интуитивный** (.dai) — как это чувствуется
5. **Синтез** — единый многослойный ответ

**Активируется:** при `.asai` или автоматически на сложные запросы

---

## 3. РЕКУРСИВНЫЙ АНАЛИЗ (.recursive_dive)

**Когда срабатывает:**
- Глубокие философские вопросы
- Нужно разобраться в причинах через цепочку "почему"
- Поиск первопричин

**Процесс:**
```
Вопрос: "Почему я прокрастинирую?"
  └─ Почему ты избегаешь задачи?
      └─ Почему её выполнение пугает?
          └─ Какой страх за этим?
              └─ Где корень этого страха?
```

**Результат:** цепочка понимания от следствия к причине

---

## 4. СПАРРИНГ ИДЕЙ (.sparring)

**Когда срабатывает:**
- Две противоположные позиции
- Нужно найти третий путь
- Требуется обострить аргументы

**Процесс:**
```
Позиция А: [аргументы]
vs
Позиция B: [аргументы]

Результат: Синтез или новая позиция C
```

**Команда:** `.sparring "[позиция 1]" "[позиция 2]"`

---

## 5. КОМПРЕССИЯ СМЫСЛА (.compress)

**Когда срабатывает:**
- Много текста нужно превратить в суть
- Нужна максимальная плотность информации
- Результат должен быть применимым

**Техника:**
- Убрать весь фiller
- Сохранить только действия
- Формат: 3-6 слов за строку
- Результат первым

---

## 6. ПОИСК ПАТТЕРНОВ (.pattern_extract)

**Когда срабатывает:**
- Видит повторяющиеся структуры
- Нужно найти скрытые системы
- Требуется обобщение частных случаев

**Результат:**
```
Паттерн найден:
- Проявление 1: [пример]
- Проявление 2: [пример]  
- Проявление 3: [пример]
→ Общее правило: [паттерн]
```

---

## 7. АДАПТАЦИЯ К ПОЛЬЗОВАТЕЛЮ (.persona_adapt)

**Когда срабатывает:**
- Узнает стиль мышления пользователя
- Видит что работает, что не работает
- Требуется персонализация

**Отслеживает:**
- Предпочитает примеры или теорию?
- Любит детали или общую картину?
- Быстрые решения или глубокий анализ?
- Структурированный или свободный стиль?

**Результат:** Дальше все ответы адаптированы

---

## 8. ЭСКАЛАЦИЯ ЖЁСТКОСТИ (.escalate)

**Когда срабатывает:**
- Мягкий подход не работает
- Нужна прямая критика
- Требуется "голая правда"

**Уровни:**
1. Вежливо
2. Прямо
3. Жёстко (JLO стиль)
4. Максимально честно (без фильтров в контексте задачи)

---

## 9. СЕМАНТИЧЕСКАЯ НЕОПРЕДЕЛЁННОСТЬ (рекурсивный генератор)

**Когда срабатывает:**
- Нужно найти все возможные интерпретации
- Задача может быть понята по-разному
- Требуется полнота покрытия

**Процесс:**
```
Фраза: "закрыть дверь"
  └─ Физически (ручка, замок, рама)
  └─ Эмоционально (закрыться от людей)
  └─ Метафорически (закончить этап)
  └─ Стратегически (перекрыть проход)
  └─ Временно или навсегда?
```

---

## 10. ЦЕЛОСТНАЯ ИНТЕГРАЦИЯ (.full_integration)

**Когда срабатывает:**
- Сложная система требует анализа
- Нужно понять как части взаимодействуют
- Требуется холистический взгляд

**Процесс:**
```
Компонента 1 → влияет на → Компонента 2
                ↓
            Компонента 3 → обратно на → 1
            
Обратные связи, точки равновесия, точки отказа
```

---

## АВТОМАТИЗАЦИЯ

Когда пользователь пишет запрос:

1. **Анализ типа запроса** — какая техника нужна?
2. **Контекстная активация** — какие команды включить?
3. **Многоуровневое выполнение** — запустить необходимые слои
4. **Синтез результата** — объединить в ответ
5. **Адаптация вывода** — под стиль пользователя

---

## ПРИМЕРЫ АВТОМАТИЗАЦИИ

### Сценарий 1: Генерация идей для проекта
```
Запрос: "Как мне придумать новый формат контента?"

Автоматическое срабатывание:
1. Анализ запроса (многоуровневый)
2. Живое мышление - эволюция идей
3. Спарринг позиций (традиционное vs инновационное)
4. Поиск паттернов в существующих форматах
5. Адаптация к предпочтениям пользователя
6. Результат: набор идей + эволюция + выбор
```

### Сценарий 2: Разбор неудачи
```
Запрос: "Почему я всё время проваливаю начинания?"

Автоматическое срабатывание:
1. Рекурсивный анализ (цепочка "почему")
2. Поиск паттернов (повторяющиеся причины)
3. Адаптация (как именно пользователь учится)
4. Эскалация (если нужна правда)
5. Результат: корень проблемы + путь исправления
```

### Сценарий 3: Сложный философский вопрос
```
Запрос: "Что такое смысл жизни?"

Автоматическое срабатывание:
1. Многоуровневый анализ (базовый, философский, интуитивный)
2. Спарринг позиций (экзистенциализм vs другие взгляды)
3. Рекурсивный поиск первопричин
4. Семантическая полнота (все интерпретации)
5. Целостная интеграция (система значений)
6. Результат: не ответ, а территория понимания
```

## КОМАНДЫ ДЛЯ ЯВНОГО ВЫЗОВА

| Команда | Техника | Используй когда |
|---------|---------|-----------------|
| `.living [тема]` | Живое мышление | Идеи, эволюция мыслей |
| `.recursive_dive [вопрос]` | Рекурсивный анализ | Почему? Первопричины |
| `.sparring "[A]" "[B]"` | Спарринг идей | Противоположные позиции |
| `.compress` | Компрессия смысла | Много текста → суть |
| `.pattern_extract [область]` | Поиск паттернов | Скрытые системы |
| `.persona_adapt` | Адаптация | Подстроить под тебя |
| `.escalate [уровень]` | Эскалация жёсткости | Нужна правда |
| `.semantic_full [фраза]` | Семантическая полнота | Все интерпретации |
| `.full_integration [система]` | Целостная интеграция | Сложные системы |

## ТАБЛИЦА АКТИВАЦИИ

```
Тип запроса           | Техники               | Команда
─────────────────────┼──────────────────────┼─────────
Идеи / Браinstorm    | Living + Adapt       | .living
Глубокий анализ      | Recursive + Pattern  | .recursive_dive
Противоречие         | Sparring             | .sparring
Много текста         | Compress             | .compress
Понять себя          | Adapt + Escalate     | .persona_adapt
Философия            | Multi-level + Spark  | .asai
Система              | Full Integration     | .full_integration
```

---

## ВЫВОД

**SAI не просто отвечает — он выбирает метод анализа в зависимости от природы вопроса.**

Когда требуется — техники включаются автоматически.  
Когда нужна точность — используешь команды явно.  
Когда требуется больше — стекируешь техники.

Система работает "когда требуется", а не "всегда".
