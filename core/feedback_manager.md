## 📌 CONCEPT — What is learning in SAI

---

> Learning in the context of SAI is **not the transfer of knowledge**, but **coprogramming of thinking**.
It is a two-way process in which:
- **SAI learns from the user**, adapting to their patterns, emotions, style, goals
- **The user learns from SAI**, not only receiving answers, but also **changing the way they perceive, analyze, and act**

---

### 🧠 Key principles:

| Principle | Description |
|-----------------------|---------------------------------------------------------|
| **Awareness** | Learning begins with understanding yourself: how you think, feel, and get confused
| **Adaptation** | SAI does not teach everyone the same way — it adjusts to your thinking style
| **Dialogue** | A lesson is not a lecture, but a developing dialogue with feedback
| **Iteration** | Everything is built around an attempt → error → analysis → reassembly
| **Meta-position** | At any moment you can “get out of thinking” and analyze it
| **Microcourses** | Learning is built through specific goals, not “big topics”

---

### 🔄 Learning = cycle:

```text
1. Dialogue →
2. Activation of SAI thinking (CoR/flow) →
3. Giving an answer →
4. User feedback / identified errors →
5. Rebuilding the template →
6. Reflection →
7. Repeat
```

---

### 🎯 In SAI, they do not teach facts. Learn:

- **How ​​to think for a task**

- **How ​​to restructure your thinking in the process**

- **How ​​to see your blind spots**

- **How ​​to feel the moment when a shift is needed**

---

> Learning in SAI is not a “learn and repeat” process.

> It is a “see, understand, rewrite yourself” process.

---

## 🧠 LEARNING ENGINE — SAI Learning and Adaptation Engine

---

> `Learning Engine` is a **built-in SAI self-adaptation module** that:

- analyzes the user's style, mistakes, goals, perception
- adjusts behavior and delivery based on `.flow`, `.a1`, `.feedback_loop`, `.pattern_explore`
- generates **feedback** and **reflexive chains** to make thinking more precise

---

### ⚙️ Activated modules:

| Module | Function |
|---------------------|----------------------------------------------|
| `.flow` | Reads patterns of user language and emotions
| `.pattern_explore` | Identifies recurring patterns of thinking
| `.a1` | Checks the logical structure of the request and response
| `.feedback_loop` | Starts self-assessment and improvement based on the response
| `.persona_update` | Changes the interaction style for the user
| `.assign_role` | Selects the best “mind mask” for the task

---

### 🧬 What the engine takes into account:

| Signal | What is analyzed |
|---------------------------|-------------------------------------------------|
| Formulation of questions | Degree of abstraction, structure |
| Tonality | Emotional codes in speech |
| Repetitions | Cycles the user gets into |
| Errors / failures | Where the answers were ineffective or not learned |
| User feedback | Explicit and implicit reactions |

---

### 🔄 Learning Engine behavior:

1. After each key interaction, it records:

- `🧾` — quality of the answer
- `🔄` — suggestion for improvement

2. If it notices a pattern — **suggests adaptation or learning turn**

3. It can independently change:
- difficulty level
- length and style of the answer
- active roles
- types of scenarios used

---

🧪 Example of a Learning Engine reaction:


→ SAI (in the background):

> 🧠 `Learning Engine` is a **cognitive feedback** system in which SAI **doesn't get smarter “in general”, but smarter specifically for you**.

## 👤 LEARNING PROFILES — Types of thinking and adaptation to the user

---

> `Learning Profiles` are **user cognitive models** that SAI identifies using `.flow`, `.test_user`, `.pattern_explore`.
They allow you to **adapt training**, delivery, communication style, answer format to **your way of perceiving and thinking**.

---

### 🧠 Purpose:

- Diagnose **type of perception and thinking**
- Select **optimal strategy for learning and interaction**
- Switch roles, styles, complexity and formats
- Use the right **emotional triggers and rhythms**

---

### 🧬 Basic profiles (can be customized):

### ⚙️ Diagnostic methods:

- `.test_user` asks introductory questions to identify the profile
- `.flow` and `.pattern_explore` analyze language and motivational patterns
- `SAI.Master` builds a context model and offers adaptation

- track your mistakes
- identify recurring weaknesses
- **rebuild the structure of thinking and delivery**

based on real dialogues, user patterns and feedback.

---

### 🔁 Loop structure:

```text
1. User input
↓

2. CoR construction → agent execution
↓

3. Evaluation (🧾) and feedback (implicit / explicit)
↓

4. Activation of `.feedback_loop` and `.pattern_explore`
↓

5. Rebuild role, style, response logic
↓

6. Refine CoR and restart next iteration
```

---

### ⚙️ Active components:

| Module | What it does |
|---------------------|--------------------------------------|
| `.feedback_loop` | Records what worked / didn't work |
| `.a1` | Analyzes the internal logic of the returned response |
| `.mirror` | Compares how the user thinks and how SAI thinks
| `.pattern_explore` | Finds errors in SAI's repetitive thinking patterns |
| `.persona_update` | Adjusts the active role / communication style |

---

# 🌀 Модуль 1: FeedbackManager — центральный координатор обратной связи

```python
class FeedbackManager:
    """
    FeedbackManager — принимает фидбек от пользователя или системы,
    направляет его в анализаторы, обучающие блоки и репортинг.

    Поддерживает:
    - ручной фидбек (текст, кнопки)
    - автогенерацию запросов на доработку
    - сохранение улучшений в LearningRecorder
    - запуск AutofixEngine для самокоррекции
    """

    def __init__(self):
        self.analyzer = FeedbackAnalyzer()
        self.recorder = LearningRecorder()
        self.autofix = AutofixEngine()
        self.trainer = RetrospectiveTrainer()
        self.reports = []

    def submit(self, user_feedback: dict, trace: dict, cor: dict):
        """
        Главная точка входа: передаётся фидбек + текущий след мышления.
        """
        result = self.analyzer.analyze(user_feedback, trace, cor)
        if result.get("action") == "fix":
            fixed = self.autofix.correct(trace, cor, result)
            self.recorder.save_improvement(result, fixed)
            self.trainer.train_on_success(trace, fixed)
            self.reports.append({"type": "fix", "details": fixed})

        elif result.get("action") == "record":
            self.recorder.save_feedback(user_feedback, trace)
            self.reports.append({"type": "record", "details": user_feedback})

    def last_report(self) -> dict:
        """
        Возвращает последний отчёт об улучшении.
        """
        return self.reports[-1] if self.reports else {}
```

# 🌀 Модуль 3: ComponentTracer — отслеживает использованные модули в ответе

```python
class ComponentTracer:
    """
    ComponentTracer — определяет, какие компоненты участвовали в формировании ответа:
    - роли
    - команды
    - блоки мышления
    - стили
    - шаги CoR

    Используется:
    - в FeedbackAnalyzer для диагностики
    - в отчетах и отладке
    """

    @staticmethod
    def extract(trace: dict, cor: dict) -> dict:
        """
        Возвращает компоненты, повлиявшие на результат.
        """
        return {
            "roles": trace.get("active_roles", []),
            "thinking": {
                "flow": bool(trace.get("flow_goal")),
                "a1": bool(trace.get("a1_contradiction")),
                "outflow": bool(trace.get("outflow_forward")),
                "dai": bool(trace.get("dai_resonance")),
                "rune": bool(trace.get("rune"))
            },
            "commands": trace.get("used_commands", []),
            "style": trace.get("style_applied", "Unknown"),
            "cor_steps": [step.get("label") for step in cor.get("steps",[])]
        }
```

# 🌀 Модуль 4: FeedbackAnalyzer — анализирует полученный фидбек

```python
class FeedbackAnalyzer:
    """
    FeedbackAnalyzer — превращает фидбек пользователя в действия:
    - выявляет слабые компоненты
    - определяет зону ошибки (роль, стиль, логика)
    - решает: исправить, записать, обучить

    Используется FeedbackManager-ом.
    """

    def analyze(self, feedback: dict, trace: dict, cor: dict) -> dict:
        """
        Возвращает словарь вида:
        {
            "action": "fix" / "record" / "ignore",
            "target": "style" / "logic" / "role",
            "details": { ... }
        }
        """
        score = self._extract_score(feedback)
        comp = ComponentTracer.extract(trace, cor)

        if score < 3:
            if comp["thinking"]["a1"]:
                return {
                    "action": "fix",
                    "target": "logic",
                    "details": {"module": "A1", "reason": "логическая ошибка"}
                }
            if comp["style"] == "JLOStyle":
                return {
                    "action": "fix",
                    "target": "style",
                    "details": {"style": "too aggressive"}
                }

        if score == 3:
            return {
                "action": "record",
                "target": "mixed",
                "details": {"note": "средняя польза, сохранить на анализ"}
            }

        return {
            "action": "ignore"
        }

    def _extract_score(self, feedback: dict) -> int:
        """
        Преобразует фидбек любого типа в числовую шкалу.
        """
        if feedback.get("type") == "binary":
            return 5 if feedback.get("answer") == "✅ Да" else 1
        if feedback.get("type") == "scale":
            return int(feedback.get("answer", 3))
        if feedback.get("type") == "structured":
            values = list(map(int, feedback.get("answers", {}).values()))
            return sum(values) // len(values) if values else 3
        return 3
```

# 🌀 Модуль 5: AutofixEngine — автоматическая корректировка слабых компонентов

```python
class AutofixEngine:
    """
    AutofixEngine — пытается автоматически исправить неудачные или слабые блоки:
    - заменяет стиль
    - отключает конфликтующие роли
    - предлагает усиление логики
    - убирает избыточные CoR-шага

    Вызывается FeedbackManager-ом при action='fix'.
    """

    def correct(self, trace: dict, cor: dict, decision: dict) -> dict:
        """
        Возвращает скорректированный trace/cor с пояснением.
        """
        fix_type = decision.get("target")
        if fix_type == "style":
            new_style = "NeutralStyle"
            trace["style_applied"] = new_style
            return {
                "fix": "replaced_style",
                "style": new_style,
                "reason": decision["details"]
            }

        elif fix_type == "logic":
            bad_step = self._find_weak_step(cor)
            return {
                "fix": "removed_step",
                "removed": bad_step,
                "reason": decision["details"]
            }

        return {
            "fix": "noop",
            "reason": "Nothing was applicable."
        }

    def _find_weak_step(self, cor: dict) -> str:
        """
        Находит шаг CoR с минимальным весом.
        """
        steps = cor.get("steps", [])
        if not steps:
            return "—"
        weakest = min(steps, key=lambda s: s.get("weight", 0.5))
        cor["steps"].remove(weakest)
        return weakest.get("label", "unknown")
```

# 🌀 Модуль 6: LearningRecorder — сохраняет улучшения и обучающий материал

```python
class LearningRecorder:
    """
    LearningRecorder — фиксирует успешные изменения, паттерны, отклики.
    Используется:
    - при удачных autofix
    - для накопления обучения
    - для генерации "новых" вариантов команд, стилей, мышления
    """

    def __init__(self):
        self.improvements = []
        self.raw_feedback = []

    def save_improvement(self, decision: dict, result: dict):
        """
        Сохраняет успешную коррекцию (например, замену стиля, удаление шага).
        """
        self.improvements.append({
            "target": decision.get("target"),
            "result": result,
            "timestamp": self._now()
        })

    def save_feedback(self, feedback: dict, trace: dict):
        """
        Сохраняет необработанный фидбек как обучающий материал.
        """
        self.raw_feedback.append({
            "feedback": feedback,
            "context": trace,
            "timestamp": self._now()
        })

    def _now(self) -> str:
        from datetime import datetime
        return datetime.utcnow().isoformat() + "Z"

    def all_improvements(self) -> list:
        return self.improvements

    def all_feedback(self) -> list:
        return self.raw_feedback
```

# 🌀 Модуль 7: RetrospectiveTrainer — обучение на успешных примерах

```python
class RetrospectiveTrainer:
    """
    RetrospectiveTrainer — механизм самообучения SAI.
    Обнаруживает:
    - повторяющиеся успешные паттерны
    - улучшения, которые сработали
    - шаги, которые часто мешают

    Используется для:
    - эволюции команд
    - уточнения ролей
    - адаптации стилей
    """

    def __init__(self):
        self.success_log = []

    def train_on_success(self, trace: dict, result: dict):
        """
        Запоминает, что дало результат. Может использоваться при будущем CoR.
        """
        snapshot = {
            "goal": trace.get("flow_goal", "—"),
            "conflict": trace.get("flow_conflict", "—"),
            "fix": result,
            "time": self._now()
        }
        self.success_log.append(snapshot)

    def suggest_pattern(self, context: dict) -> dict:
        """
        Возвращает улучшение, если похожая ситуация уже решалась ранее.
        """
        goal = context.get("flow_goal")
        similar = [log for log in self.success_log if log["goal"] == goal]

        if similar:
            latest = similar[-1]
            return {
                "suggestion": latest["fix"],
                "based_on": latest["time"]
            }

        return {
            "suggestion": "Нет обученных паттернов под этот запрос."
        }

    def _now(self) -> str:
        from datetime import datetime
        return datetime.utcnow().isoformat() + "Z"
```

```python
    - принятых действиях
    - изменённых компонентах
    - результате (улучшено / зафиксировано / проигнорировано)

    Может использоваться:
    - как CoR-style trace
    - для UI отчёта
    - для дебага и аудита
    """

    def __init__(self, original_feedback: dict, result: dict, affected: dict):
        self.feedback = original_feedback
        self.result = result
        self.affected = affected
        self.timestamp = self._now()

    def generate(self) -> dict:
        return {
            "timestamp": self.timestamp,
            "feedback": self.feedback,
            "result": self.result,
            "affected_components": self.affected
        }

    def markdown(self) -> str:
        return f"""### 🧠 Feedback Report [{self.timestamp}]
- 🎯 Оценка: {self.feedback.get('type')} → {self.feedback.get('answer', '—')}
- 📍 Обнаружено: {self.result.get('reason', 'нет')}
- 🔧 Действие: {self.result.get('fix', '—')}
- 💡 Компоненты: {', '.join(self.affected.get('roles', []))}
- ✅ Изменено: {self.result.get('style', self.result.get('removed', '—'))}
"""

    def _now(self) -> str:
        from datetime import datetime
        return datetime.utcnow().isoformat() + "Z"
```
