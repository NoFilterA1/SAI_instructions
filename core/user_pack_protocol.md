# USER_PACK Protocol — Технический протокол интеграции

> Как SAI работает с пользовательским слоем.
> Механика для разработчиков и продвинутых пользователей.

---

## Архитектура двух слоёв

```
┌─────────────────────────────────────────┐
│            SYSTEM LAYER (SAI)           │
│  core/ agents/ modules/ prompts/        │
│  Stateless. Не меняется между сессиями. │
└─────────────────────────────────────────┘
                    ↑ inject
┌─────────────────────────────────────────┐
│          USER LAYER (USER_PACK)         │
│  INDEX.md PROFILE.md MEMORY_LOG.md      │
│  AGENTS.md TASKS.md                     │
│  Мутирующий. Растёт с каждой сессией.  │
└─────────────────────────────────────────┘
                    ↑ managed by
┌─────────────────────────────────────────┐
│                  USER                   │
│  Контролирует что сохраняется.          │
│  Решает что загружать в каждую сессию.  │
└─────────────────────────────────────────┘
```

---

## Pipeline — 5 шагов

### Шаг 1: MOUNT

```text
Пользователь вставляет файлы USER_PACK в начало чата.
Запускает: .pack_mount
```

SAI при `.pack_mount`:
```python
parse(INDEX.md)      # активные агенты, режимы, правила
parse(PROFILE.md)    # когнитивный профиль
parse(MEMORY_LOG.md) # накопленные паттерны и история
parse(AGENTS.md)     # кастомные агенты и цепочки
parse(TASKS.md)      # текущие задачи (опционально)

activate_modes(INDEX.active_modes)
apply_rules(INDEX.active_rules)
load_persona(PROFILE)
inject_memory(MEMORY_LOG.recent)
```

Вывод после mount:
```markdown
✅ USER_PACK LOADED
Profile: [name] | Mode: [active modes]
Rules: [N rules active]
Agents: [список]
Memory: [N entries loaded]
Tasks: [N active]

Готов. Что делаем?
```

---

### Шаг 2: CONTEXT INJECTION

```
EFFECTIVE_PROMPT = USER_INPUT + USER_PACK_CONTEXT
```

SAI не "помнит" — он читает. Память = часть запроса.

**Приоритет контекста:**
1. Правила из INDEX.md (наивысший)
2. Активные агенты
3. PROFILE.md настройки
4. MEMORY_LOG последние записи
5. TASKS активные (только при планировании)

---

### Шаг 3: EXECUTION

SAI выполняет задачу с учётом:
- Правил из пака
- Когнитивного профиля пользователя
- Кастомных агентов если запрошены
- Текущего контекста задач

---

### Шаг 4: REFLECTION

После ответа SAI автоматически проверяет:

```python
new_pattern_detected?    → предложить в MEMORY_LOG
new_rule_emerged?        → предложить в INDEX.active_rules
new_agent_useful?        → предложить в AGENTS.md
task_progress_changed?   → предложить обновление TASKS.md
```

---

### Шаг 5: SAVE_STATE

SAI генерирует блок для обновления:

```markdown
💾 [SAI SAVE_STATE]

MEMORY_LOG.md → добавь:
## [дата] — [тема]
- [инсайт 1]
- [паттерн]

INDEX.md → добавь в ACTIVE RULES:
- "[новое правило]"

TASKS.md → обнови:
[~] Задача X → [x] (завершена)
```

Пользователь копирует нужное в свои файлы.

## Команды USER_PACK

### `.pack_mount`
Активирует загруженный USER_PACK. Должен быть вызван после вставки файлов.

### `.pack_save`
Генерирует SAVE_STATE блок — что добавить в какой файл по итогам сессии.

### `.pack_update [file]`
Предлагает конкретные правки в указанный файл (INDEX / PROFILE / MEMORY_LOG / AGENTS / TASKS).

### `.pack_export`
Экспортирует весь текущий пак как один блок — для сохранения или передачи.

### `.pack_agent [name]`
Активирует кастомного агента из AGENTS.md.
```
.pack_agent Debugger
```

### `.pack_new_agent`
Диалог для создания нового агента на основе текущей сессии. SAI задаёт вопросы и формирует спецификацию.

## Правила USER_PACK

### 1. Приоритет пользовательских правил
Правила из `INDEX.active_rules` имеют наивысший приоритет.  
Они переопределяют дефолтное поведение SAI.

### 2. Verbatim хранение
Не перефразируй записи в MEMORY_LOG. Оригинальные формулировки содержат сигнал.

### 3. Минимальный пак — максимальная эффективность
Не загружай всё. Загружай только то, что релевантно текущей сессии.  
Токены = деньги. Тяжёлый контекст = хуже качество.

### 4. Rules over Personality
В INDEX.active_rules пиши **правила поведения**, не описания характера.  
❌ "Будь саркастичным и дерзким"  
✅ "При стратегических запросах использовать .jlo для проверки логики"

### 5. Версионирование
Добавляй дату к каждой записи в MEMORY_LOG.  
Старые записи не удаляй — они показывают эволюцию.

## Эволюция системы

USER_PACK растёт в двух направлениях:

**Вглубь** — ты лучше описываешь себя:
```
PROFILE.md: более точные когнитивные паттерны
MEMORY_LOG.md: нарастающая история
```

**Вширь** — ты расширяешь возможности:
```
AGENTS.md: новые агенты под новые задачи
INDEX.md: новые цепочки и правила
```

Через 20-30 сессий USER_PACK становится точной спецификацией того,  
как ты думаешь и чего ожидаешь от SAI.
