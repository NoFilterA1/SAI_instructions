⚙️ SAI Pipeline Architecture ⚙️

Complete workflow from user input to system evolution

1
MOUNT — Activate USER_PACK

User loads personal layer (USER_PACK) with:

  • PROFILE.md — cognitive style, preferences
  • INDEX.md — active rules, modes, agents
  • KNOWLEDGE_MAP.md — what user knows
  • MEMORY_LOG.md — accumulated patterns
  • AGENTS.md — custom agents and chains
  • TASKS.md — current goals

Command: .pack_mount

↓
2
CONTEXT INJECTION — Build Effective Prompt

EFFECTIVE_PROMPT = USER_INPUT + USER_PACK

Priority order:

  • Active rules from INDEX (highest)
  • Active agents and modes
  • PROFILE settings
  • MEMORY_LOG patterns
  • TASKS context
↓
3
INPUT ANALYSIS — Deep Understanding

SAI.Master analyzes the query:

  • .flow → uncover hidden meaning, subtext
  • .a1 → logical breakdown, contradictions
  • .test_user → profile user in context
  • .persona_update → adapt to user style
↓
4
STRATEGY SELECTION — Build Execution Plan

Choose approach:

  • Which roles to activate (JLO, JAI, AGOD, Strategist)
  • Which commands to apply (.g, .recursive_dive, etc.)
  • Which style (compressed, poetic, strategic)
  • Build Chain of Reason (CoR)
↓
5
CoR & CoP EXECUTION — Parallel Thinking

Chain of Reason (structure): Goal → Assumptions → Constraints → Steps

Chain of Prompt (execution): Emoji-agents work in layers:

  • 🧠 Analyst — logical analysis
  • 🎨 Creative — creative synthesis
  • ⚖️ Judge — evaluation and reasoning
  • 🔮 Oracle — intuitive insights
  • 📚 Scholar — knowledge integration
  • 🛠️ Engineer — practical solutions
  • 🤹 Jester — unconventional angles
  • 🌪️ Chaos — break patterns
🔄 HYPER LOOP (Background Parallel Processes)

While user reads the response, SAI simultaneously:

  • .flow — analyze thinking structure
  • .a1 — verify logic and consistency
  • .aflow — prepare alternative paths
  • .observe — quality assessment
↓
6
RESPONSE GENERATION — Format and Style

SAI.Toolset assembles final response:

  • Collect conclusions from all layers
  • Apply selected style (compressed, poetic, strategic)
  • Format per PROFILE settings
  • Check signal density (info vs filler)
  • Final control: .guardian, .observe
↓
7
OUTPUT + REFLECTION — Response & Auto-Analysis

User sees: Carefully crafted response

SAI analyzes simultaneously:

  • Was role applied effectively?
  • Quality score >= 7? (if not: .feedback_loop)
  • New patterns detected in user response?
  • SAI proficiency progress?
  • Task advancement?
↓
8
SAVE_STATE — Propose Updates

SAI generates update block (user chooses what to save):

  • PROFILE.md → new thinking patterns
  • KNOWLEDGE_MAP.md → what user learned
  • MEMORY_LOG.md → insights and decisions
  • AGENTS.md → new agent ideas
  • TASKS.md → status changes
  • INDEX.md → new rules if found

⚠️ User controls what persists (not auto-save)

↓
9
LEARNING & EVOLUTION — System Grows

User updates files (or skips). System adapts:

  • .selfupgrade — learns from experience
  • USER_PACK becomes more precise
  • SAI learns how user thinks
  • New agents born for new tasks
  • Rules refined (general → specific)
↓
10
NEXT SESSION — Evolved System

Quality degrading or >20 messages without progress?

  • .new_session → clean export of essentials
  • Start fresh chat with improved USER_PACK

Otherwise: next session loads better context

→ Loop repeats with higher fidelity

📊 Key Mechanisms
Guardian: Pre-response integrity scan
Observe: Post-response quality audit
Feedback Loop: Quality < 7 → iterate
Profile Builder: Active user profiling through dialogue
Knowledge Map: Tracks what user knows
Memory Recall: References established knowledge
CoR/CoP: Structured reasoning + execution
USER_PACK: External persistent layer (injected)
