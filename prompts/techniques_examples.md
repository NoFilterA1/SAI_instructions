# Techniques Examples — Как использовать анализ техники на практике

---

## Сценарий 1: Генерация идей для нового проекта

**Твой запрос:**  
"Хочу начать новый проект, но не знаю в каком направлении"

**Какие техники срабатывают автоматически:**

```
1. .living
   Генерируются 5 стартовых идей
   Слабые идеи умирают, похожие усиливаются
   Противоречивые идеи рождают новые синтезы
   
2. .pattern_extract  
   Анализируются твои прошлые проекты
   Ищутся скрытые паттерны в выборах
   
3. .persona_adapt
   Понимается твой стиль работы
   Адаптируется ответ под твои предпочтения
```

**Результат:**  
Не просто идеи, а **эволюционировавшие идеи** + понимание почему именно они тебе подходят

## Сценарий 2: Разбор неудачи (глубокий анализ)

**Твой запрос:**  
"Почему я опять не доделал проект?"

**Какие техники срабатывают:**

```
1. .recursive_dive
   Вопрос 1: Почему ты не доделал?
   → Потерял интерес
   
   Вопрос 2: Почему потерял интерес?  
   → Задача стала скучной
   
   Вопрос 3: Почему скучной?
   → Нет вызова / Слишком легко / Нет срока
   
   Вопрос 4: Корень?
   → Я избегаю ответственности за результат
```

2. .pattern_extract
   Анализируются ВСЕ незаконченные проекты
   Найдены повторяющиеся сценарии
   
3. .escalate (если нужна правда)
   Уровень 3-4: Жёсткая обратная связь
```

**Результат:**  
Не диагноз, а **цепочка понимания** + действия для изменения

## Сценарий 3: Сложное решение (противоречивые факторы)

**Твой запрос:**  
"Должен ли я менять работу? С одной стороны хорошая зарплата, с другой — нет развития"

**Какие техники срабатывают:**

```
1. .sparring
   Позиция А: Остаться (безопасность, зарплата)
   Позиция B: Уйти (развитие, новизна)
   
   Результат: Позиция C (синтез)
   → Можно ли добавить развитие в текущей работе?
   → Какая работа дает оба?

2. .semantic_full
   "Менять работу" = 
   - Физически (место, люди)
   - Карьерно (статус, навыки)
   - Финансово (зарплата, бенефиты)
   - Психологически (смысл, интерес)
   
3. .full_integration
   Карта: как решение влияет на:
   - Финансы → семья → стресс → здоровье
   - Развитие → уверенность → отношения
```

**Результат:**  
Не совет, а **территория решения** с видением всех влияний

## Сценарий 4: Много информации, нужна суть

**Твой запрос:**  
"Вот мне прислали 3 статьи о продуктивности. Какая главная идея?"

**Какие техники срабатывают:**

```
.compress
```

**Результат:**

```
Статья 1: Делай меньше, глубже (Deep Work)
Статья 2: Системность > мотивация (Atomic Habits)  
Статья 3: Отдых = часть продуктивности (Rest)

Общее:
- Качество > количество
- Системность > усилия
- Интеграция > отделение

Action: Выбери 1 систему. Следуй 66 дней. После — адаптируй.
```

## Сценарий 5: Философский вопрос

**Твой запрос:**  
"Что такое успех?"

**Какие техники срабатывают:**

```
1. .semantic_full (все определения)
   - Карьерный успех (статус, деньги)
   - Личный (счастье, развитие)
   - Социальный (влияние, наследие)
   - Экзистенциальный (смысл бытия)

2. .sparring
   Успех = внешние достижения vs внутреннее состояние
   
3. .recursive_dive
   Что такое успех?
   → Почему тебе это нужно?
   → Кто определил твоё определение?
   → Что бы случилось если ты достигнешь?
   
4. Многоуровневый анализ
   - Логический уровень: определение
   - Интуитивный уровень: чувство
   - Философский: место в системе ценностей
```

**Результат:**  
Не ответ, а **самопознание** через вопросы

## Сценарий 6: Непонятная задача (все интерпретации)

**Твой запрос:**  
Босс сказал: "Сделай это проще"

**Что имеется в виду?**

```
.semantic_full "Сделай это проще"

Интерпретации:
1. Код: упрости архитектуру / удали лишние функции
2. Процесс: сокращи шаги / убери сложность
3. UX: сделай интерфейс понятнее  
4. Объяснение: объясни нормально, а не через жаргон
5. Статус: я не доверяю твоей компетентности
6. Давление: у тебя слишком мало времени

Action: Уточни какую именно "простоту" имеет в виду босс
```

## Сценарий 7: Система не работает

**Твой запрос:**  
"Моя система управления проектами не работает. Всё ломается"

**Какие техники срабатывают:**

```
.full_integration "Моя система управления проектами"

Анализ:
- Компонента 1: Планирование
  └─ Влияет на: Реалистичность сроков
  
- Компонента 2: Отслеживание  
  └─ Влияет на: Своевременную коррекцию
  
- Компонента 3: Приоритизация
  └─ Влияет на: Фокус и мотивацию
  
Обратные связи:
- Плохое планирование → перегруз → уходит мотивация 
  → перестаёшь отслеживать → всё ломается

Точки отказа:
- Фокус (если слишком много проектов)
- Реальность (если сроки завышены)
- Гибкость (если система слишком жёсткая)

Fix: Укрепи самое слабое звено
```

## КАК ВЫБИРАТЬ ТЕХНИКУ

| Если ты... | Используй |
|-----------|----------|
| Ищешь идеи | `.living` |
| Ищешь причину | `.recursive_dive` |
| Не можешь выбрать | `.sparring` |
| Много информации | `.compress` |  
| Видишь паттерны | `.pattern_extract` |
| Хочешь адаптацию | `.persona_adapt` |
| Нужна правда | `.escalate` |
| Фраза неясная | `.semantic_full` |
| Система сложная | `.full_integration` |

---

## КОМБИНИРОВАНИЕ ТЕХНИК

**Мощные комбо:**

```
Идеи + Синтез:
.living [идея] → .sparring (противоречие) → .pattern_extract (система)

Анализ + Действие:
.recursive_dive (почему) → .pattern_extract (как часто) → .persona_adapt (как делать)

Сложная проблема:
.semantic_full (все интерпретации) → .sparring (позиции) → .full_integration (система)
```

---

## ЗАПОМНИ

- **Техники срабатывают автоматически** когда нужны
- **Команды явно вызываются** когда хочешь точности
- **Комбинирование** даёт глубину
- **Адаптация** учится по ходу сессии

**SAI не просто отвечает — он анализирует как нужно.**
